# Decision Log: 2026-06-01

## DL-001: Claude readiness baseline

### Participants
- Human: project owner
- AI: ChatGPT

### Question
What must be fixed before using Claude to implement the full SettlePass MVP?

### Decision
Prioritize documentation, API contracts, demo fixtures, data model, and module decisions before business-logic implementation.

### Rationale
The existing wireframe is suitable for demo UI, but Claude needs stricter boundaries to avoid:
- storing forbidden identity data
- mixing Clean Architecture layers
- treating mockTxHash as a real transaction
- expanding Phase 1 into Sui/zkLogin/Walrus/PTB
- implementing backend modules without clear module invariants

### Resulting changes
- Add `CLAUDE.md`
- Add AI orchestration docs
- Expand architecture/API/security/data model docs
- Add module docs for core backend modules
- Add Phase 1 shared fixtures
- Expand API contract types
- Expand Prisma schema draft
